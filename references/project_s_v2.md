# Project S — 체대 정시 SaaS 기획서 v2

**버전:** v2 (기획 확장판)  
**용도:** 내부 팀 정렬 + PM/개발 인계  
**상태:** 기획 단계  
**서비스명:** Project S (가칭)  
**변경:** 가능  
**범위:** 정시 + 실기 + 전략  
**타겟:** 체대 정시 준비 수험생 (B2C)

## 0. 개요

Project S는 체대 정시 준비 과정에서 필요한 **수능 성적 입력 → 실기 데이터 입력 → 군별 전략 조합 → 대학 선택 → 총점 확인** 과정을 지원하는 SaaS 서비스이다.

본 기획서는 **기획 단계 기준** 문서이며, 설계/DB/아키텍처 단계는 후속 단계에서 별도 진행한다.

## 1. 시장 정의

체대 정시 시장은 다음 요소로 구성됨:

- 정시는 **군 기반 선택 시장 (가/나/다)**
- 수능 성적 + 실기 성적의 **복합 점수 반영**
- 대학별 반영 방식이 상이함
- 전략 의사결정이 존재함

## 2. 사용자 정의 (Persona)

**주 사용자:** 체대 정시 수험생 본인  
**보조 사용자:** 학부모, 학원, 컨설턴트 (선택적)

사용자 상태:

- 수능 성적 보유 (모의 → 본수능)
- 실기 기록 보유
- 지원 대학 탐색
- 군 전략 고려
- 총점 확인 및 비교 필요

## 3. 연간 입시 캘린더

체대 정시는 **9개월 사용 → 5개월 피크 사용** 패턴을 가짐

```
2월 ~ 5월 : 실기 준비 + 모의 성적 입력
6월 : 6월 모의 반영
7월 ~ 8월 : 실기 집중 + 대학 탐색
9월 : 9월 모의 반영 (전략 중요)
10월 ~ 12월 : 실기 최종 + 전략 조합
1월 1~4주 : 실기 시험 + 원서 접수
2월 1~3주 : 발표 및 충원
```

## 4. 사용자 작업 흐름 (Behavior Flow)

사용자는 다음 단계로 진행:

```
1. 성적표 입력
2. 군 검색
3. 대학 검색
4. 과 검색
5. 수능 환산 확인
6. 실기 입력
7. 총점 확인
```

## 5. 입력 데이터 정의 (명시)

### 5.1 수능 성적표 입력

사용자는 **성적표 그대로 입력**한다.

```
수능 성적표 입력

- 국어: (표점 / 백분위 / 등급)
   → 선택과목 존재 (27학년도까지 유지)
- 수학: (표점 / 백분위 / 등급)
- 탐구1: (표점 / 백분위 / 등급)
- 탐구2: (표점 / 백분위 / 등급)
- 영어: 등급
- 한국사: 등급
- 선택과목 정보
- 시험 종류 (3월 / 6월 / 9월 / 본수능)
```

**표준점수/백분위/등급 중 어떤 수치를 사용할지는 대학별로 상이하며, 사용자는 입력만 하며 환산은 시스템 내에서 처리한다.**

### 5.2 가채점 단계 입력

```
가채점 입력 단계
입력: 원점수만 입력
출력: 표점 / 백분위 / 등급 → 시스템이 유추
```

목적: 수시 전략 시뮬레이션

### 5.3 성적표 배부 대응

- 성적표 배부일 기준 입력 요청
- 비입력 시 경고 표시
- 모의고사/본수능 동일 구조

### 5.4 시험 종류

사용자는 입력 시 시험 종류 선택:

```
(3월 / 6월 / 9월 / 본수능)
```

## 6. 실기 입력

사용자는 실기 기록을 입력함.

입력항목 예시:

- 제자리멀리뛰기
- 배근력
- 메디신볼던지기
- 사이드스텝
- 기타 대학 요구 종목

입력 후 총점 계산은 대학 반영 방식에 따라 시스템 내부에서 처리.

## 7. 전략 요소

사용자는 다음 전략 요소를 고려:

- 군 선택 (가/나/다)
- 대학 선택
- 학과 선택
- 실기 비중
- 수능 반영 방식

Project S는 해당 전략을 지원하기 위한 구조를 제공함.

## 8. 출력

출력되는 최종 결과:

- 수능 환산 점수
- 실기 점수
- 총점
- 대학별 비교
- 군별 조합 비교

## 9. 데이터 유지 요소

연간 업데이트 요소:

- 대학별 전형 반영식
- 최고표점 업데이트 (6월/9월 수능 반영 참고)
- 실기 반영 방식
- 가산/감점 조건
- 모집 군 배치
- 모집단위 변화
- 실기 종목 변경
- 실기 배점 변경

## 10. 사용 기간 목표

Project S는 단일 시즌 사용이 아닌

**2월 → 9월 유지 → 10~2월 피크**

구조를 목표로 함.
